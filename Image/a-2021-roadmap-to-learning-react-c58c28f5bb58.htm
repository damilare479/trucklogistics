A 2021 Roadmap to Learning React

All the things you need to know to go from beginner to expert

React is one of the most popular JavaScript libraries for creating dynamic web applications. It has huge community support and an active development team behind it.

Although React by itself is relatively simple, it’s not enough to just learn a single library, especially if you want to create a complex web app.

I’ve been developing in React since 2016, and as my tasks increased in complexity, I had to learn other “helper libraries” to implement features.

In this article, I want to lay out a roadmap you can use for learning how to create real-world applications using React. I will list the important React concepts and other useful libraries you should know.

Prerequisites

Since React is a JavaScript library, you should know JavaScript basics and ES6 syntax really well.

I would recommend that you only start to learn React after you can code a simple website using pure HTML/CSS and Vanilla JavaScript.

Basics

In this section, I want to share things that you must learn in order to build a simple but functional React application. The recommended resources for learning (and ones that I’ve used) will be listed at the end of the section.

create-react-app

facebook/create-react-app

Create React apps with no build configuration. Create React App works on macOS, Windows, and Linux. If something…

github.com

I believe in learning by coding, and this library will help you to get started with a React application in no time.

It will set up a project with some default files and let you dive straight into coding and see how React works.

If you are adventurous, you can also eject the app and get deeper into how React code is transpiled for it to run on the browser. But if you are a beginner, you shouldn’t be worried about how to create a React app from scratch just yet.

JSX

You should start with the most basic concepts of React, which is creating components using JSX. You will rarely see a real-world application written without it. Get familiar with the following topics:

  • Rendering an element using JSX
  • Embedding JavaScript expressions
  • Specifying element attributes
  • Programmatically updating an element and seeing it change in the browser

Components

This is a very important concept to grasp. A React application is composed of many standalone and reusable components.

Components are pieces of code that have high specificity and are usually isolated. Each component handles one thing really well, and data are shared through props or context.

For example, you might have a single header component that is only responsible for rendering navigation links.

There are four parts to this concept:

  • Learning how to communicate data between components
  • Training your mind to think of a webpage in terms of components
  • Lifecycle methods and states
  • Function vs. class components

You should grasp the concept of props, how to pass them down to child components, and how to verify props using PropTypes. Context is another way of communicating data between components, but it’s more of an advanced concept.

Then you’ll want to start thinking of web pages in terms of components. Take any web page, like H&M’s official website, and try to break it down into components. This will force you to focus on the smallest possible bits of code and to practice code design. I was also asked to do this multiple times in an interview.

Each component has a set of lifecycle methods, and dynamic data are stored in states. When data stored in state changes, the component will re-render to update that change. These are fundamental concepts to understand as well.

When learning these concepts, you will undoubtedly run into conditional rendering and rendering multiple components from a list. At this point, you should be able to create a simple React application.

The final point to understand is the differences between functional and class components and their use cases. This will lead right into Hooks.

Hooks

Introducing Hooks - React

Hooks are a new addition in React 16.8. They let you use state and other React features without writing a class. This…

reactjs.org

React Hook is a recent addition to the library in React 16.8. It’s used with function components, and it allows developers to use state and other features without using a class.

Before, functional components were stateless. States and lifecycle methods were used with class components. With Hooks, developers can use state with functional components.

You should know how to use the most common Hooks, like useState and useEffect.

React Router

React Router: Declarative Routing for React

Learn once, Route Anywhere

reactrouter.com

React Router is the router library for React, and it allows you to render different components based on the URL.

Learning this library will be the first step for creating a full-stack React app.

React Query

React Query

Instead of writing reducers, caching logic, timers, retry logic, complex async/await scripting (I could keep going...)…

react-query.tanstack.com

A web app must be able to fetch data, and complex ones have lots of states to maintain. React doesn’t have an opinionated way of fetching and updating data, and managing states is hard. So libraries like Redux were born.

However, Redux is complicated and introduces lots of boilerplate code. If you are just starting to learn React, you should go with an easier option to get started.

React Query is another lightweight library, which is often described as “the missing data-fetching library for React.” It’s a lot easier to learn than Redux, and it can utilize Hooks (which you should be familiar with now).

This will make data fetching really easy and give you a taste of how real-world applications do it. (I used this library extensively at work.)

More Advanced

Redux and Redux Thunk

github.com

Despite being complicated and introducing a lot of boilerplate code for even the most simple data fetch, Redux is still a very popular and widely used state management library in the industry.

You should learn how to do the following:

  • Set up a project with Redux
  • Use Redux for data fetching and managing state
  • Optionally, look into Redux Thunk

I had a hard time understanding Redux in the beginning. Here is an article that really explained all the concepts in detail, with a step-by-step tutorial on Redux.

Redux Thunk is a popular library often used together with Redux. It allows action creators to return functions instead of an action object. It’s optional for learning React but still a good library to know.

Testing

React Testing Library | Testing Library

React Testing Library builds on top of DOM Testing Library by adding APIs for working with React components. Projects…

testing-library.com

Writing tests is important, and in my experience, it’s usually harder than writing actual code.

You should learn the most popular testing libraries, like Jest and Enzyme, and how to mock API calls with libraries like Sinon. There are other libraries, like React Testing Library, as well.

As your application gets complicated, it’s important to have good unit tests for individual components to ensure correctness, instead of assuming they are behaving as intended.

Advanced React concepts

React official documentation conveniently separates all the advanced concepts from the main ones. Technically, you don’t need to know any of the advanced concepts to build a React application.

However, production-level applications often utilize some of these concepts, such as refs and higher-order components. It’s always useful to fully understand what React is capable of, even if you don’t use it.

Most of these concepts you might stumble upon when writing code. It would be useful to at least skim through this section to understand what React is fully capable of.

How React works under the hood

At this stage, you are ready to dive into the inner workings of React.

You can first run eject on a React app created by create-react-app to understand how React code is transpiled to run on the browser.

You can also try to set up a React application yourself without using the create-react-app library. Look into Gulp, webpack, or Babel and see how you can transpile React code using these libraries. A lot of applications require more flexibility in the setup than that provided by create-react-app.

Lastly, you can look into React’s source code to get a full understanding of the library.

Final Thoughts

Hopefully, this list can be helpful for you, whether you are just a beginner trying to learn React or you want to freshen up your React knowledge.

React has been actively developed by Facebook, and over the years many new features and libraries have emerged to support this amazing and easy-to-use library.

I’m very excited about what’s yet to come, and I hope you are as well.

Finally, I want to wish you all the best on your React journey!
